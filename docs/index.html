gmail-cli

A stateless, zero-dependency, agent-friendly Gmail CLI. Read, send, reply, forward, label, and organize mail straight from the shell — JSON on every command.

$npx @hartewired/gmail-cli help

Stateless by design

Each command mints or reuses a cached token, calls the Gmail API, prints JSON, and exits.

Why another Gmail CLI

Most Gmail tools are built for a human at a terminal. This one is built to be driven by a program — an LLM agent, a cron job, a shell script.

  1. Stateless. No daemon, no background auth server, no long-running process to crash or reconnect. Each command mints or reuses a cached access token and exits. Nothing runs when idle.
  2. Zero dependencies. Pure Node (>= 20, built-in fetch). Nothing to audit, nothing to break on an upstream release.
  3. Machine-readable. JSON by default on write and organize commands means output parses cleanly. Search-query batch ops (--query) let one command act on many messages.

Want a rich human TUI? Use himalaya. Want something an agent can shell out to and parse? That's this.

The shape of it

One-time OAuth sign-in against your own Google Cloud client, then every command just works.

# one-time sign-in (loopback captures the redirect; --manual for headless boxes)
gmail auth

# list, read, act — Gmail search syntax everywhere
gmail list "is:unread newer_than:3d" --max 10 --json
gmail send --to a@b.com --subject "Q2 report" --html --body-file note.html --attach report.pdf
gmail reply <id> --all --body "Thanks — got it."
gmail markread --query "is:unread older_than:30d"
